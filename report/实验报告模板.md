# 基于 FCN 的 PASCAL VOC 五类语义分割实验报告

## 1. 实验目的

本实验基于全卷积网络（Fully Convolutional Network, FCN）完成多类别语义分割任务，理解像素级分类、上采样与跳跃连接的作用；通过 `FCN+ResNet18` 和 `FCN+ResNet34` 的对比，分析编码器深度对精度、训练效率与模型大小的影响；掌握 mIoU 和像素准确率的计算与解读方法。

## 3. 实验原理

### 3.1 语义分割与 FCN

语义分割要求为图像中的每个像素分配语义类别。本实验输出背景、人、车、猫、狗五个类别。FCN 将分类网络中的全连接层替换为卷积预测层，使网络可以输出空间分布的类别得分图，再通过上采样恢复到输入分辨率。

本项目采用 FCN-8s 风格结构，从 ResNet 编码器的 `layer2`、`layer3`、`layer4` 提取多尺度特征。深层特征提供语义信息，浅层跳跃连接补充空间细节，最终输出与输入同尺寸的五类 logits。

### 3.2 数据集与标签设计

实验使用 PASCAL VOC 2012 官方语义分割 `train`/`val` 划分。在每个划分内筛选至少出现一个目标前景类别的图像，不跨越官方训练与验证边界。

| 类别 | VOC 原始 ID | 本实验 ID |
| --- | ---: | ---: |
| Background | 0 | 0 |
| Person | 15 | 1 |
| Car | 7 | 2 |
| Cat | 8 | 3 |
| Dog | 12 | 4 |
| 其他类别及标注边界 | 其余 / 255 | 255（忽略） |

损失和指标计算均排除标签为 `255` 的像素，避免把不属于目标分类范围的物体错误作为背景学习。

### 3.3 评价指标

像素准确率：

```text
Pixel Accuracy = 正确预测的有效像素数 / 全部有效像素数
```

类别交并比：

```text
IoU_c = TP_c / (TP_c + FP_c + FN_c)
```

平均交并比：

```text
mIoU = mean(IoU_background, IoU_person, IoU_car, IoU_cat, IoU_dog)
```

像素准确率容易受到大面积背景像素影响；mIoU 更能反映各类别的平衡分割性能。

## 4. 实验步骤

1. 在服务器安装与 CUDA 适配的 PyTorch、torchvision 和项目依赖。
2. 下载或放置 VOC2012 数据集，执行 `python prepare_data.py --root ./data` 检查子集规模。
3. 使用相同训练策略分别训练 `FCN-ResNet18` 与 `FCN-ResNet34`。
4. 加载两者验证集 mIoU 最优权重，计算 Pixel Accuracy、mIoU、分类别 IoU、权重文件大小与推理耗时。
5. 汇总对比表，选择验证图像生成分割可视化结果。

## 7. 结果分析

### 7.1 模型对比分析

待根据真实结果填写。建议说明以下内容：

- ResNet34 相比 ResNet18 的 mIoU 是否提升，以及提升主要出现在哪些类别；
- 更深编码器增加的参数量、权重大小和训练/推理时间；
- Pixel Accuracy 与 mIoU 变化是否一致，背景占比较高是否掩盖前景类别错误。

### 7.2 可视化分析

待根据生成图片填写。可从以下方面分析：

- 目标较大、边界清晰、背景简单的样本是否预测更完整；
- 小目标、遮挡、目标边界或复杂背景区域是否存在漏分和错分；
- 猫与狗等形态相近类别是否存在混淆；
- FCN 双线性上采样输出的边界是否较粗糙。

### 7.3 局限性与改进方向

经典 FCN 结构能够完成密集预测，但对目标边界细节和多尺度场景的处理能力有限。后续可尝试转置卷积上采样、不同编码器、类别加权损失，或与 DeepLab 系列模型比较。
