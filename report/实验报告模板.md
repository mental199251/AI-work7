# 基于 FCN 的 PASCAL VOC 五类语义分割实验报告

## 1. 实验目的

本实验基于全卷积网络（Fully Convolutional Network, FCN）完成多类别语义分割任务，理解像素级分类、上采样与跳跃连接的作用；通过 `FCN+ResNet18` 和 `FCN+ResNet34` 的对比，分析编码器深度对精度、训练效率与模型大小的影响；掌握 mIoU 和像素准确率的计算与解读方法。

## 3. 实验原理

### 3.1 语义分割与 FCN

语义分割要求为图像中的每个像素分配语义类别。本实验输出背景、人、车、猫、狗五个类别。FCN 将分类网络中的全连接层替换为卷积预测层，使网络可以输出空间分布的类别得分图，再通过上采样恢复到输入分辨率。

本项目采用 FCN-8s 风格结构，从 ResNet 编码器的 `layer2`、`layer3`、`layer4` 提取多尺度特征。深层特征提供语义信息，浅层跳跃连接补充空间细节，最终输出与输入同尺寸的五类 logits。

### 3.2 数据集与标签设计

实验使用 PASCAL VOC 2012 官方语义分割 `train`/`val` 划分。在每个划分内筛选至少出现一个目标前景类别的图像，不跨越官方训练与验证边界。

| 类别 | VOC 原始 ID | 本实验 ID |
| --- | ---: | ---: |
| Background | 0 | 0 |
| Person | 15 | 1 |
| Car | 7 | 2 |
| Cat | 8 | 3 |
| Dog | 12 | 4 |
| 其他类别及标注边界 | 其余 / 255 | 255（忽略） |

损失和指标计算均排除标签为 `255` 的像素，避免把不属于目标分类范围的物体错误作为背景学习。

### 3.3 评价指标

像素准确率：

```text
Pixel Accuracy = 正确预测的有效像素数 / 全部有效像素数
```

类别交并比：

```text
IoU_c = TP_c / (TP_c + FP_c + FN_c)
```

平均交并比：

```text
mIoU = mean(IoU_background, IoU_person, IoU_car, IoU_cat, IoU_dog)
```

像素准确率容易受到大面积背景像素影响；mIoU 更能反映各类别的平衡分割性能。

## 7. 结果分析

### 7.1 模型对比分析

ResNet34 的 Pixel Accuracy 为 94.35%，比 ResNet18 提高 1.79 个百分点；mIoU 为 79.85%，提高 6.22 个百分点。mIoU 的增幅明显大于像素准确率，说明更深的编码器并非仅改善大面积背景，而是改善了前景类别的均衡预测效果。

分类别看，Cat IoU 和 Dog IoU 分别提升 9.38 与 9.68 个百分点，提升最明显。这与猫、狗具有形态变化大、类别外观相似、边界复杂等特点相符。代价是 ResNet34 的模型大小从 42.69 MB 增至 81.28 MB，训练耗时约为 ResNet18 的 2.03 倍；单图推理耗时从 2.12 ms 增至 2.57 ms，增加相对有限。

### 7.2 可视化分析

`sample_0000` 中多人遮挡较明显，两种模型均能识别主要人体区域，但边界存在粘连；ResNet34 的离散错误区域更少。`sample_0014` 中三只狗姿态相互遮挡，ResNet18 出现较多错误类别区域，ResNet34 能形成更连续的狗区域预测。`sample_0028` 中猫和狗同时出现，两者均能区分类别，ResNet34 对局部边界的处理更完整。

可视化中的灰色区域表示五类任务中被设为 `ignore_index=255` 的 VOC 非目标类别或标注边界，这些区域不参与 mIoU 和 Pixel Accuracy 计算。因此包含火车、公共汽车等大面积灰色目标的样本只适合说明忽略标签机制，不宜直接用于评价五类模型分割质量。

### 7.3 局限性与改进方向

经典 FCN 结构能够完成密集预测，但对目标边界细节和多尺度场景的处理能力有限。扩展实验将增加 ResNet50 编码器、可学习转置卷积上采样以及 VOC 完整 21 类任务，进一步比较主干深度、恢复分辨率方式和任务规模带来的性能变化。
